Типи даних
int
immutable нескінченна точність hashable
Що таке int у Python
Python int — довільної точності (BigNum). Немає переповнення як у C/Java. Маленькі int (-5..256) — синглтони: CPython кешує їх при старті.
Під капотом: CPython зберігає int як PyLongObject — масив 30-бітних "digit". Маленькі числа — 1 digit. Великі — масив. Операції над великими числами: O(n) де n — кількість digit. 
x = 10 ** 100        # Гугол — без проблем, немає OverflowError
type(x)              # <class 'int'>
x.bit_length()       # 333

# Кешування [-5..256] — identity trick
a = 256; b = 256
a is b               # True  — один і той самий об'єкт!
a = 257; b = 257
a is b               # False — вже різні об'єкти

# Системи числення
bin(255)             # '0b11111111'
oct(255)             # '0o377'
hex(255)             # '0xff'
int('ff', 16)        # 255
int('0b1010', 2)     # 10

(255).bit_length()   # 8
(256).bit_length()   # 9
O(1)
x.bit_count()  3.10+
Кількість одиниць у бінарному представленні (popcount)
(0b10110).bit_count()  # 3
O(1)
x.to_bytes(n, order)
int → bytes. order: 'big' або 'little'. Для мережевих протоколів.
(1024).to_bytes(2, 'big')    # b'\x04\x00'
(1024).to_bytes(4, 'little') # b'\x00\x04\x00\x00'
int.from_bytes(b, order)
bytes → int. Класметод.
int.from_bytes(b'\x04\x00', 'big')
# 1024
// і % нюанс: floor division завжди до -∞: -7 // 2 == -4 (не -3). Остача: -7 % 2 == 1 (знак знаменника, не діленого). Це відрізняється від C/Java!
pow(a, b, mod) набагато швидше ніж (a**b) % mod для криптографії — використовує fast modular exponentiation O(log b).
float
immutable IEEE 754 double ≈15–17 знаків
Що таке float
64-бітний double precision: 1 знак + 11 exp + 52 мантиса. Не всі десяткові дроби представимі точно у двійковій системі.
Під капотом: 0.1 у binary — 0.0001100110011... (нескінченний дріб). IEEE 754 округлює → накопичується похибка. float у CPython — C double всередині PyObject. 
0.1 + 0.2              # 0.30000000000000004 ← класика!
0.1 + 0.2 == 0.3       # False

# Правильне порівняння:
import math
math.isclose(0.1 + 0.2, 0.3)          # True
math.isclose(a, b, rel_tol=1e-9)      # з точністю

# Для грошей — Decimal:
from decimal import Decimal
Decimal('0.1') + Decimal('0.2')        # Decimal('0.3') — точно!
Спеціальні значення
float('inf')            # +∞
float('-inf')           # -∞
float('nan')            # Not a Number

nan = float('nan')
nan == nan              # False! NaN ≠ NaN (за IEEE 754)
nan is nan              # True  (той самий об'єкт)
math.isnan(nan)         # True  ← так треба перевіряти

1 / 0.0                 # ZeroDivisionError (не inf!)
import math
math.inf + 1            # inf
math.inf - math.inf     # nan

(2.0).is_integer()   # True
(2.5).is_integer()   # False

Точне представлення float як дроб p/q

x.hex() / float.fromhex()
Hex repr — точний, без втрат. Корисно для серіалізації.
(0.1).hex()
# '0x1.999999999999ap-4'
float.fromhex('0x1.8p+0')  # 1.5
round(x, n)
Banker's rounding — до парного при .5! Не звичне округлення.
round(0.5)    # 0 (не 1!)
round(1.5)    # 2
round(2.5)    # 2 (не 3!)
round(1.234, 2)  # 1.23
sys.float_info містить: max (~1.8e308), min (5e-324), epsilon (~2.2e-16) — найменше число таке що 1.0 + epsilon != 1.0.
bool
підклас int immutable синглтони
bool IS-A int — чому?
Гвідо ван Россум додав bool у Python 2.3 (2002). До цього використовували 0/1. Для зворотної сумісності bool зробили підкласом int. True і False — глобальні синглтони в CPython.
isinstance(True, int)    # True ← bool справді int!
isinstance(True, bool)   # True
bool.__bases__           # (<class 'int'>,)

True + True              # 2
True * 5                 # 5
False + 1                # 1
sum([True, False, True]) # 2 (підрахунок True)

True is True             # завжди True (синглтон)
bool(1)  == True         # True
1        == True         # True
1        is True         # False! (різні об'єкти)
Truthy / Falsy значення
# Falsy — bool(x) == False:
bool(0)         # False
bool(0.0)       # False
bool(0j)        # False (complex нуль)
bool('')        # False
bool([])        # False
bool({})        # False (порожній dict)
bool(set())     # False
bool(None)      # False

# Truthy — все інше:
bool(-1)        # True (будь-яке ненульове число!)
bool(' ')       # True (пробіл — не порожній рядок)
bool([0])       # True (список з нулем — непорожній)
bool([False])   # True!
Класична помилка: -1 і будь-яке від'ємне — truthy! Якщо функція повертає -1 як помилку: if result: не спрацює. Треба if result != -1:
Логічні оператори — повертають значення, не bool!
# and: перший falsy АБО останній (якщо всі truthy)
0 and 'hello'      # 0
1 and 'hello'      # 'hello'
[] and [1,2]       # []

# or: перший truthy АБО останній
0 or 'hello'       # 'hello'
'' or [] or 0      # 0 (всі falsy → останній)
'yes' or 'no'      # 'yes'

# Паттерн default value:
name = user_input or 'Anonymous'

# not завжди повертає bool:
not 0              # True
not 'anything'     # False

# Short-circuit — правий не обчислюється якщо результат відомий:
data = get_data()
if data and data[0] == 'x':   # безпечно: data перевіряється першим
    ...
__bool__ vs __len__ протокол
class MyList:
    def __init__(self, data): self.data = data
    def __len__(self): return len(self.data)

# Python шукає: __bool__ → __len__ → True (якщо немає нічого)
bool(MyList([]))    # False (len == 0)
bool(MyList([1]))   # True  (len > 0)

class AlwaysTrue:
    def __bool__(self): return True

bool(AlwaysTrue())  # True (навіть якщо len == 0)
str
immutable Unicode hashable
Що таке str
Python 3 str — завжди Unicode. Immutable — кожна операція створює новий рядок.
Flexible string representation (PEP 393): CPython зберігає рядок у найменшому форматі: Latin-1 (1 байт/символ) якщо всі ≤ U+00FF, UCS-2 (2 байти) до U+FFFF, UCS-4 (4 байти) якщо є emoji. 'hello' займає ~50 байт, 'hello😀' — значно більше. 
# Інтернування рядків
a = 'hello'; b = 'hello'
a is b            # True  (CPython інтернує прості str)

a = 'hello world'; b = 'hello world'
a is b            # False або True — не гарантовано!

import sys
a = sys.intern('hello world')
b = sys.intern('hello world')
a is b            # True  (явне інтернування)

# НІКОЛИ не порівнюй рядки через is — тільки ==

s.find(sub, start, end)
Індекс першого входження або -1. Не кидає виключення.

'hello world'.find('o', 5) # 7
'hello world'.find('xyz')  # -1

s.index(sub)
Як find, але ValueError якщо не знайдено

'hello'.index('xyz')  # ValueError!
O(n)
s.count(sub)
Кількість входжень без перекриття
'aababab'.count('ab')  # 3
'aaaa'.count('aa')     # 2 (не 3!)

startswith() / endswith()
Приймають tuple! Перевіряють кілька варіантів одночасно.
'readme.md'.endswith(('.md','.txt'))
# True
'GET /path'.startswith(('GET','POST'))
# True
Зміна регістру
s.upper() / s.lower()
Повний Unicode. 'ß'.upper() → 'SS' (2 символи!)
'straße'.upper()   # 'STRASSE'
s.casefold()
Агресивний lower для порівняння. Краще ніж lower() для i18n.
'ß'.casefold()   # 'ss'
'ß'.lower()      # 'ß'
# Для порівняння: s1.casefold() == s2.casefold()
s.strip(chars)
chars — множина символів, не підрядок! Видаляє з обох боків.
'xxhelloxx'.strip('x')   # 'hello'
'xyxhello'.strip('xy')   # 'hello' (і x, і y)
removeprefix() / removesuffix()
Python 3.9+. Видаляє підрядок (не символи як strip).
'unhappy'.removeprefix('un')   # 'happy'
'test.py'.removesuffix('.py')  # 'test'
Розбивка та об'єднання
s.split(sep, maxsplit)
Без sep — по whitespace + видаляє порожні. З sep — ні.
'a  b  c'.split()       # ['a','b','c']
'a  b  c'.split(' ')    # ['a','','b','','c']
'a,b,c'.split(',', 1)   # ['a','b,c']
O(n)
sep.join(iterable)
Правильний спосіб конкатенації! += у циклі — O(n²), join — O(n).
'-'.join(['a','b','c'])  # 'a-b-c'
''.join(map(str, [1,2])) # '12'
# Будувати рядок:
parts = [str(x) for x in data]
result = ', '.join(parts)  # O(n)
O(n)
s.partition(sep)
Завжди 3 елементи: (before, sep, after). Якщо немає → (s, '', '').
'user@host.com'.partition('@')
# ('user', '@', 'host.com')
'no-at'.partition('@')
# ('no-at', '', '')
s.splitlines()
Розуміє \n, \r\n, \r та Unicode line endings.
'a\nb\r\nc'.splitlines()
# ['a', 'b', 'c']
Форматування рядків

isdigit / isnumeric / isdecimal — різниця!
# isdecimal — тільки 0-9 (найстрогіше)
'123'.isdecimal()   # True
'²'.isdecimal()     # False

# isdigit — додає Unicode "цифри" (верхні індекси і т.д.)

'½'.isnumeric()     # True
'Ⅻ'.isnumeric()     # True (римська XII)

memoryview — zero-copy slicing
data = bytearray(b'hello world')
mv = memoryview(data)

# Slice без копіювання даних у пам'яті!
chunk = mv[0:5]        # <memory at 0x...>
bytes(chunk)           # b'hello' (копія тільки тут)

# Можна змінювати оригінал через view:
mv[0] = ord('H')
print(data)            # bytearray(b'Hello world')

# Корисно для великих буферів (відео, мережа):
# Без memoryview: кожен slice = нова копія (O(n) пам'яті)
# З memoryview:   slice = view (O(1) пам'яті)
memoryview критичний для продуктивності при роботі з великими бінарними даними — передає дані до socket/file без зайвих копій.
list
mutable dynamic array ordered
Що таке list
list — динамічний масив вказівників на PyObject. Не LinkedList! Кожен елемент — 8-байтовий вказівник на об'єкт.
Під капотом — dynamic array з over-allocation: При append, якщо місця немає — виділяє новий масив з надлишком (~n*1.125). Тому амортизований O(1) для append. Послідовність росту: 0→4→8→16→25→35→46→58→72→88...
Основні методи
lst.append(x)
Додає в кінець. Амортизований O(1).
lst = [1, 2, 3]
lst.append(4)   # [1, 2, 3, 4]
O(1) amorт.
lst.insert(i, x)
Всі елементи правіше зсуваються — повільно для початку!
lst.insert(0, 'x')   # вставка на початок
# Для черги → collections.deque
O(n)
lst.pop(i=-1)
pop() з кінця O(1). pop(0) — O(n)!
lst.pop()     # останній — O(1)
lst.pop(0)    # перший   — O(n)!
O(1) кінецьO(n) інакше
lst.remove(x)
Видаляє ПЕРШЕ входження. ValueError якщо немає.
[1,2,2,3].remove(2)  # [1, 2, 3]
O(n)
lst.extend(iter)
+= теж extend. Не те саме що append!
lst.extend([4,5])   # [1,2,3,4,5]
lst.append([4,5])   # [1,2,3,[4,5]] ← список!
O(k)
lst.sort(key, reverse)
Timsort. In-place. Стабільний. key — функція перетворення.
words = ['banana','apple','fig']
words.sort(key=len)
# ['fig', 'apple', 'banana']
O(n log n)
lst.copy()
Shallow copy! Вкладені об'єкти — ті самі.
a = [[1,2],[3,4]]
b = a.copy()
b[0].append(9)
print(a)  # [[1,2,9],[3,4]] !
import copy; c = copy.deepcopy(a)
O(n)
lst.index(x, s, e)
Індекс першого входження. ValueError якщо немає.
[1,2,3,2].index(2)     # 1
[1,2,3,2].index(2, 2)  # 3
O(n)

tuple
immutable hashable* швидший за list
Що таке tuple
Immutable послідовність. Може бути ключем dict/set. Швидший за list для read-only. CPython кешує порожній tuple і малі tuple.
# Одноелементний — ПОТРІБНА КОМА!

t = (1,)    # tuple
t = 1,      # теж tuple (дужки необов'язкові)

# Unpacking
x, y = 1, 2
x, y = y, x          # swap без тимчасової змінної!

a, b, *rest = (1, 2, 3, 4, 5)
# a=1, b=2, rest=[3,4,5]

# Hashable якщо всі елементи hashable
hash((1, 'hello', 3.14))   # OK
hash(([1,2], 3))            # TypeError! list не hashable

# Ключ dict:
d = {(0,0): 'origin', (1,0): 'right'}
d[(0,0)]    # 'origin'

dict
mutable hash map ordered 3.7+
Що таке dict
Compact hash table (PEP 468, Python 3.6+): Два масиви: indices (sparse, для хешування) та entries (dense, key+value+hash). Load factor ~2/3 → resize. При колізії — open addressing з perturbation. Ця оптимізація зменшила розмір dict на ~25% і гарантувала порядок вставки. 
# Створення
d = {'key': 'value'}
d = dict(a=1, b=2)
d = dict([('a', 1), ('b', 2)])

# Доступ
d['key']            # KeyError якщо немає
d.get('key')        # None якщо немає
d.get('key', 0)     # 0 якщо немає
Методи dict
d.get(key, default)
Безпечний доступ. Не кидає KeyError.
{'a':1}.get('z', 0)   # 0
O(1)
d.setdefault(key, def)
Якщо ключ є — повертає value. Якщо немає — встановлює і повертає.
d = {}
d.setdefault('a', []).append(1)
d.setdefault('a', []).append(2)
# {'a': [1, 2]}
O(1)
d.pop(key, default)
Видаляє і повертає. З default — без KeyError.
d.pop('key')          # повертає value
d.pop('missing', None) # None, без помилки
O(1)
d.popitem()
Видаляє і повертає ОСТАННІЙ вставлений (LIFO з 3.7+).
d = {'a': 1, 'b': 2}
d.popitem()   # ('b', 2)
O(1)
d.update(other)
Merge. Перезаписує існуючі ключі. 3.9+: d1 | d2 оператор.
d.update({'c': 3})
d.update(x=4)
new = d1 | d2    # новий dict (3.9+)
d1 |= d2         # in-place (3.9+)
O(k)
d.keys() / values() / items()
View-об'єкти! Динамічно відображають поточний стан dict.
d = {'a': 1}
kv = d.keys()
d['b'] = 2
list(kv)   # ['a','b'] ← оновилось!
O(1) отримання
defaultdict, Counter, comprehension
from collections import defaultdict, Counter

# defaultdict — авто-ініціалізація
dd = defaultdict(list)
dd['key'].append(1)   # не треба ініціалізувати!

dd = defaultdict(int)
for word in 'hello world hello'.split():
    dd[word] += 1
# defaultdict(int, {'hello': 2, 'world': 1})

# Counter — підрахунок елементів
c = Counter('abracadabra')
c.most_common(3)      # [('a',5), ('b',2), ('r',2)]
c['a']                # 5
c['z']                # 0 (не KeyError!)
c + Counter('abc')    # об'єднання підрахунків

# Dict comprehension

Reference counting та GC

x = [1, 2, 3]
sys.getrefcount(x)   # 2 (x + аргумент функції)

sys.getrefcount(x)   # 3

sys.getrefcount(x)   # 2

# Циклічні посилання — GC їх збирає
a = []
a.append(a)          # a вказує на себе!
del a                # refcount не падає до 0 → потрібен GC

gc.collect()         # примусовий збір циклів
gc.disable()         # вимкнути GC (обережно!)
CPython: Reference counting (основний) + Cyclic GC для циклічних посилань. PyPy використовує tracing GC. __del__ (finalizer) викликається при refcount → 0, але не гарантовано при циклах.
