🚀 Початок роботи
$git init
Ініціалізує новий Git-репозиторій у поточній папці. Створює прихований каталог .git.
ПРИКЛАДgit init
З НАЗВОЮgit init my-project
$git clone
Клонує існуючий репозиторій з віддаленого сервера на локальну машину.
HTTPSgit clone https://github.com/user/repo.git
SSHgit clone git@github.com:user/repo.git
ПАПКАgit clone URL my-folder
$git config
Налаштування імені та пошти автора. Записується у кожен коміт.
ІМ'Яgit config --global user.name "Іван"
ПОШТАgit config --global user.email "i@mail.com"
СПИСОКgit config --list
📝 Базові зміни
$git status
Показує поточний стан: змінені, нові та непроіндексовані файли.
ПРИКЛАДgit status
КОРОТКОgit status -s
$git add
Додає файли до індексу (staging area) для наступного коміту.
ОДИН ФАЙЛgit add index.html
ВСІgit add .
ТИПgit add *.js
$git commit
Зберігає проіндексовані зміни як новий знімок (snapshot) у історії.
З ПОВІД.git commit -m "feat: додати форму входу"
ADD + COMMITgit commit -am "fix: виправити баг"
ВИПРАВИТИgit commit --amend -m "нове повідомлення"
$git diff
Показує різницю між версіями файлів.
НЕЗАІНДЕКС.git diff
ЗАІНДЕКС.git diff --staged
ГІЛКИgit diff main..feature
$git log
Виводить історію комітів з автором, датою та повідомленням.
БАЗОВОgit log
КОМПАКТНОgit log --oneline
ГРАФgit log --oneline --graph --all
$git stash
Тимчасово ховає поточні зміни, щоб переключитися на іншу задачу.
ЗБЕРЕГТИgit stash
ДІСТАТИgit stash pop
СПИСОКgit stash list
🌿 Гілки (Branches)
$git branch
Управління гілками: список, створення, видалення.
СПИСОКgit branch
СТВОРИТИgit branch feature/login
ВИДАЛИТИgit branch -d feature/login
ВСІgit branch -a
$git switch / checkout
Переключення між гілками. switch — новий спосіб (рекомендовано).
НА ГІЛКУgit switch main
НОВА+ПЕРЕЙТИgit switch -c feature/auth
СТАРИЙgit checkout -b feature/auth
$git merge
Зливає зміни з іншої гілки в поточну. Об'єднує історію комітів.
ПРИКЛАДgit merge feature/login
БЕЗ FFgit merge --no-ff feature/login
--no-ff зберігає запис про злиття гілки
$git rebase
Переносить коміти поточної гілки на вершину іншої. Дає лінійну історію.
ПРИКЛАДgit rebase main
ІНТЕРАКТ.git rebase -i HEAD~3
-i дозволяє редагувати, squash, drop коміти
☁️ Робота з Remote
$git remote
Управління віддаленими репозиторіями.
СПИСОКgit remote -v
ДОДАТИgit remote add origin URL
ЗМІНИТИgit remote set-url origin NEW_URL
$git fetch / pull
Отримання змін з remote. fetch — тільки завантажує, pull — завантажує і зливає.
FETCHgit fetch origin
PULLgit pull origin main
REBASEgit pull --rebase origin main
$git push
Відправляє локальні коміти на віддалений репозиторій.
БАЗОВОgit push origin main
НОВА ГІЛКАgit push -u origin feature/ui
ТЕГИgit push --tags
↩️ Скасування змін
$git restore
Скасовує зміни у файлах або видаляє їх зі staging area (замінює частину checkout).
ФАЙЛgit restore index.html
З ІНДЕКСУgit restore --staged index.html
$git revert
Створює новий коміт, який скасовує зміни попереднього. Безпечно для shared-гілок.
ПРИКЛАДgit revert abc1234
ОСТАННІЙgit revert HEAD
⚠️ НЕБЕЗПЕЧНО — ЗМІНЮЄ ІСТОРІЮ
$git reset
Скасовує коміти, переміщуючи HEAD назад. Ніколи не використовуй на публічних гілках!
SOFTgit reset --soft HEAD~1
— скасує коміт, зміни залишаться в індексі
HARDgit reset --hard HEAD~1
— ВИДАЛИТЬ зміни повністю без повернення
🏷️ Теги та корисні команди
$git tag
Мітить конкретний коміт — зазвичай для релізів (v1.0.0).
СПИСОКgit tag
ANNOTATEDgit tag -a v1.0.0 -m "Release 1.0"
ВІДПРАВИТИgit push origin v1.0.0
$git cherry-pick
Переносить конкретний коміт з іншої гілки в поточну.
ПРИКЛАДgit cherry-pick abc1234
Корисно для хотфіксів між гілками
$git bisect
Бінарний пошук коміту, який вніс баг. Дуже потужний інструмент дебагінгу.
СТАРТgit bisect start
ПОГАНИЙgit bisect bad
ХОРОШИЙgit bisect good v1.0
$.gitignore
Файл зі списком патернів файлів, які Git має ігнорувати.
ВМІСТnode_modules/
ВМІСТ*.env
ВМІСТdist/ .DS_Store
gitignore.io — генератор для будь-якого стеку
