list — динамічний масив
Під капотом

list — це dynamic array поверх C-масиву покажчиків (PyObject*). CPython зберігає масив покажчиків на об'єкти, не самі об'єкти. Кожен елемент — 8 байт (покажчик на 64-bit системі).

При переповненні capacity зростає за формулою over-allocation: new_cap ≈ n + n>>3 + (3 або 6)

Послідовність росту: 0 → 4 → 8 → 16 → 25 → 35 → 46 → 58...
Коефіцієнт ~1.125x, а не 2x як у більшості мов!
sys.getsizeof([]) = 56 байт (заголовок). Кожен слот = 8 байт. list з 1000 int-ів займає ~8 КБ покажчиків, але int-объекти займають окрему пам'ять.
Складність операцій
l[i]O(1) — прямий зсув у C-масиві
l.append(x)O(1) amortized, O(n) при resize
l.pop()O(1) — декрементуємо size
l.pop(i)O(n) — зсув всіх елементів вліво
l.insert(i, x)O(n) — зсув вправо від i
l.remove(x)O(n) — лінійний пошук + зсув
x in lO(n) — sequential scan
l.sort()O(n log n) — Timsort
l.reverse()O(n) — swap покажчиків in-place
l.extend(it)O(k) де k = len(it)
l[a:b]O(k) — копіює k=b-a покажчиків
l * nO(n·k) — shallow copy!
Timsort — чому він геніальний

Timsort (Tim Peters, 2002) = merge sort + insertion sort. Шукає вже відсортовані "runs" у реальних даних. Insertion sort для runs менше 64 елементів. Гарантований O(n log n), але O(n) для вже впорядкованих.

# Два sort = стабільна multi-key сортировка
# (Schwartzian transform)
data.sort(key=lambda x: x[0])  # secondary
data.sort(key=lambda x: x[1])  # primary зберігається!

# key= використовується один раз per element
# (не перевиклика при кожному порівнянні)
data.sort(key=lambda x: x[1])  # O(n) key calls + O(n log n) compares

Під капотом — open addressing

set / frozenset — hash table без values
Під капотом

set реалізований як hash table без values (тільки keys). Та сама відкрита адресація що і dict. Внутрішньо: setentry = структура з полями hash (Py_hash_t) + key (PyObject*).

Мінімальний розмір = 8 слотів. Load factor < 2/3. При resize × 4.

x in set = O(1). x in list = O(n). Для дедублікації та перевірки наявності — завжди set. 1000 елементів: list ~1мс, set ~100нс.

s.add(x)O(1) avg
s.remove(x)O(1) avg, KeyError якщо нема
s.discard(x)O(1) avg, без помилки
s.pop()O(1) — довільний елемент
s | t (union)O(len(s) + len(t))
s & t (intersect)O(min(len(s), len(t)))
s - t (diff)O(len(s))
s ^ t (sym_diff)O(len(s) + len(t))
s.issubset(t)O(len(s))
s.issuperset(t)O(len(t))
s.isdisjoint(t)O(min(len(s),len(t)))

deque — doubly-linked list of blocks
Під капотом

collections.deque — двозв'язний список фіксованих блоків (fixed-size blocks). Кожен блок = 64 елементи. Блоки зв'язані між собою. Не переміщує елементи при insert/pop з обох кінців.

list.insert(0, x) = O(n) — зсуває всі елементи. deque.appendleft(x) = O(1). Для черг і стеків — deque ЗАВЖДИ краще list!
Складність операцій
d.append(x)O(1)
d.appendleft(x)O(1)
d.pop()O(1)
d.popleft()O(1)
d[0], d[-1]O(1)
d[i] (середина)O(n) — обхід блоків!
d.rotate(k)O(k)
d.extend(it)O(k)
x in dO(n)
len(d)O(1)
d[i] — O(n) на відміну від list! deque НЕ є заміною list для random access.

from collections import deque

# BFS — класичне використання
def bfs(graph, start):
    queue = deque([start])
    visited = {start}
    while queue:
        node = queue.popleft()   # O(1)!
        for nbr in graph[node]:
            if nbr not in visited:
                visited.add(nbr)
                queue.append(nbr)

heapq — binary min-heap поверх list
Під капотом

heapq реалізує binary min-heap поверх звичайного Python list. Heap property: heap[k] <= heap[2k+1] та heap[k] <= heap[2k+2]. Елемент heap[0] завжди мінімальний.

# Структура в пам'яті (list під капотом):
#        1
#       / \
#      3   2
#     / \ / \
#    7  5 4  6
# [1, 3, 2, 7, 5, 4, 6]

# heap[k] = parent of heap[2k+1] і heap[2k+2]
heapify(lst) = O(n), НЕ O(n log n)! Це нетривіальний математичний факт. heappush кожен раз = O(n log n) загалом.
Складність операцій
heapify(lst)O(n) — математичне диво!
heappush(h, x)O(log n) — sift up
heappop(h)O(log n) — sift down
heappushpop(h,x)O(log n) — атомарно!
heapreplace(h,x)O(log n) — pop потім push
nsmallest(k,it)O(n log k)
nlargest(k,it)O(n log k)
h[0] (peek)O(1) — мінімум без видалення
Просунуті паттерни

# Медіан потоку (two heaps trick)
lo = []  # max-heap (зберігаємо negated)
hi = []  # min-heap

def add_num(num):
    heapq.heappush(lo, -num)
    heapq.heappush(hi, -heapq.heappop(lo))
    if len(lo) < len(hi):
        heapq.heappush(lo, -heapq.heappop(hi))
generators — lazy evaluation
Як працюють генератори

Generator function повертає generator object — реалізує протокол ітератора (__iter__ + __next__). При yield виконання призупиняється, стан (frame) зберігається.

Під капотом: generator = Python frame object + стан виконання. Frame зберігає локальні змінні, стек та поточну інструкцію (lasti).

def gen():
    print("start")
    yield 1         # призупиняється тут
    print("resume")
    yield 2         # призупиняється тут

g = gen()
next(g)   # виводить "start", повертає 1
next(g)   # виводить "resume", повертає 2
next(g)   # StopIteration
yield from — делегування
# yield from = делегує до sub-generator
# Підтримує send() та throw() через generator chain
def chain(*iterables):
    for it in iterables:
        yield from it

# Coroutine через send()
def accumulate():
    total = 0
    while True:
        value = yield total
        if value is None: break
        total += value

acc = accumulate()
next(acc)        # ініціалізація! повертає 0
acc.send(10)     # 10
acc.send(20)     # 30
acc.send(5)      # 35
acc.send(None)   # StopIteration

def read_lines(files):
    for f in files: yield from open(f)

    for line in lines: yield json.loads(line)

    for r in records:

def controlled():
    try:
        while True:
            x = yield
            print(f"Got: {x}")

GC / Memory — як Python керує пам'яттю
Reference Counting

CPython використовує reference counting як основний механізм. Кожен PyObject має поле ob_refcnt. При refcnt=0 — об'єкт звільняється НЕГАЙНО (детерміновано).

import sys
a = [1, 2, 3]
sys.getrefcount(a)  # 2 (a + аргумент функції)

b = a               # refcnt = 3
del b               # refcnt = 2
del a               # refcnt = 1 → 0 → звільняється

# Проблема: циклічні посилання
a = []
a.append(a)    # refcnt ніколи не стане 0!
# → потрібен cyclic GC
__del__ викликається при refcnt=0. Не покладайся на час виклику — з cyclic GC він недетермінований. Використовуй context managers!

class Vector:
    def __init__(self, x, y): self.x, self.y = x, y

    # Арифметика
    def __add__(self, o):  return Vector(self.x+o.x, self.y+o.y)
    def __sub__(self, o):  return Vector(self.x-o.x, self.y-o.y)
    def __mul__(self, n):  return Vector(self.x*n, self.y*n)
    def __rmul__(self, n): return self.__mul__(n)  # 3 * v
    def __neg__(self):     return Vector(-self.x, -self.y)
    def __abs__(self):     return (self.x**2 + self.y**2)**0.5

    # Порівняння (+ functools.total_ordering)
    def __eq__(self, o): return (self.x, self.y) == (o.x, o.y)
    def __lt__(self, o): return abs(self) < abs(o)
    def __hash__(self):  return hash((self.x, self.y))

    # Представлення
    def __repr__(self):  return f"Vector({self.x}, {self.y})"
    def __str__(self):   return f"({self.x}, {self.y})"
    def __bool__(self):  return bool(self.x or self.y)

    # Контейнер
    def __iter__(self):  yield self.x; yield self.y
    def __len__(self):   return 2
    def __getitem__(self, i): return (self.x, self.y)[i]

    # Callable
    def __call__(self, t): return Vector(self.x*t, self.y*t)
Descriptor protocol
# Descriptor = об'єкт що керує атрибутом іншого класу
class TypedAttr:
    def __set_name__(self, owner, name):
        self.name = name
        self.private = f"_{name}"

    def __get__(self, obj, objtype=None):
        if obj is None: return self  # клас, не екземпляр!
        return getattr(obj, self.private, None)

    def __set__(self, obj, value):
        if not isinstance(value, self.type):
            raise TypeError(f"{self.name}: expected {self.type}")
        setattr(obj, self.private, value)

class IntAttr(TypedAttr): type = int
class StrAttr(TypedAttr): type = str

class Person:
    age  = IntAttr()   # descriptor!
    name = StrAttr()

# Metaclass = клас класів
class SingletonMeta(type):
    _instances = {}

    def __call__(cls, *args, **kwargs):
        if cls not in cls._instances:
            cls._instances[cls] = super().__call__(*args, **kwargs)
        return cls._instances[cls]

class DB(metaclass=SingletonMeta): pass
DB() is DB()  # True!

# __enter__ + __exit__
class Timer:
    def __enter__(self):
        import time
        self.start = time.perf_counter()
        return self

    def __exit__(self, exc_type, exc_val, exc_tb):
        self.elapsed = time.perf_counter() - self.start
        return False  # True = suppress exception

with Timer() as t:
    result = slow_operation()
print(f"Took {t.elapsed:.3f}s")
