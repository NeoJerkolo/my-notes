Абстракція

Класи і об'єкти — основа всього

Анатомія класу
Клас — це шаблон. Об'єкт — це екземпляр шаблону. __init__ — конструктор, self — посилання на поточний екземпляр.
class Car: # Атрибут класу — спільний для всіх екземплярів wheels: int = 4 def __init__(self, brand: str, speed: int) -> None: # Атрибути екземпляра — унікальні для кожного об'єкта self.brand = brand # публічний self._speed = speed # захищений (конвенція) self.__vin = "SECRET" # приватний (name mangling) def describe(self) -> str: return f"{self.brand}, {self._speed} km/h" def __repr__(self) -> str: return f"Car(brand={self.brand!r})" car = Car("BMW", 200) print(car.describe()) # BMW, 200 km/h print(car._Car__vin) # SECRET (name mangling розкривається так) print(car.wheels) # 4 (атрибут класу) Car.wheels = 6 # зміна для ВСІХ екземплярів!
self — це перший аргумент будь-якого методу екземпляра. Python передає його автоматично при виклику car.describe(). Ім'я "self" — лише конвенція, але не міняй його.
@classmethod
Метод класу — отримує клас (cls), а не екземпляр. Використовується як альтернативний конструктор.
class Date: def __init__(self, y, m, d): self.year, self.month, self.day = y, m, d @classmethod def from_string(cls, s: str): y, m, d = map(int, s.split("-")) return cls(y, m, d) # cls() = Date() d = Date.from_string("2024-01-15")
@staticmethod
Статичний метод — не отримує ні self, ні cls. Просто функція всередині класу для логічної організації.
class MathHelper: @staticmethod def is_prime(n: int) -> bool: if n < 2: return False for i in range(2, int(n**0.5)+1): if n % i == 0: return False return True MathHelper.is_prime(17) # True
__slots__ — оптимізація пам'яті
За замовчуванням Python зберігає атрибути в словнику __dict__. __slots__ замінює словник на більш ефективну структуру — економить 40-60% пам'яті.
class Point: __slots__ = ('x', 'y') # тільки ці атрибути дозволені! def __init__(self, x: float, y: float): self.x = x self.y = y p = Point(1.0, 2.0) # p.z = 3.0 → AttributeError! тільки x і y # p.__dict__ → AttributeError! немає словника # Корисно для мільйонів об'єктів (точки, вектори, записи)
__slots__ не наслідується автоматично. Підклас без __slots__ знову матиме __dict__. Оголошуй __slots__ у кожному класі ієрархії.

Інкапсуляція — контроль доступу до даних

@property — геттери і сеттери по-пітонячому
Property дозволяє звертатись до методу як до атрибута, додаючи логіку валідації без зміни публічного API.
class Temperature: def __init__(self, celsius: float = 0): self._celsius = celsius @property def celsius(self) -> float: """Геттер — викликається при t.celsius""" return self._celsius @celsius.setter def celsius(self, value: float) -> None: """Сеттер — викликається при t.celsius = value""" if value < -273.15: raise ValueError(f"Нижче абсолютного нуля: {value}") self._celsius = value @property def fahrenheit(self) -> float: """Обчислюваний (read-only) атрибут""" return self._celsius * 9/5 + 32 @celsius.deleter def celsius(self) -> None: self._celsius = 0 # reset при del t.celsius t = Temperature() t.celsius = 100 # OK print(t.fahrenheit) # 212.0 t.celsius = -300 # ValueError!
Дескриптори — @property на стероїдах
Дескриптор — об'єкт, що реалізує __get__, __set__, __delete__. Це механізм, на якому побудований @property, classmethod, staticmethod. Використовуй для реюзабельної логіки валідації.
class Validated: """Дескриптор: перевіряє тип і діапазон значення""" def __set_name__(self, owner, name): self.name = name self.private_name = f"_{name}" def __get__(self, obj, objtype=None): if obj is None: return self # доступ через клас return getattr(obj, self.private_name, None) def __set__(self, obj, value): if not isinstance(value, (int, float)): raise TypeError(f"{self.name} має бути числом") if value < 0: raise ValueError(f"{self.name} має бути ≥ 0") setattr(obj, self.private_name, value) class Product: price = Validated() # дескриптор застосовується quantity = Validated() # до кожного атрибута окремо def __init__(self, price, quantity): self.price = price self.quantity = quantity p = Product(9.99, 100) p.price = -1 # ValueError!
Протокол дескриптора: якщо об'єкт має __get__ — він non-data descriptor (як функції). Якщо має __get__ і __set__ — data descriptor (вищий пріоритет над __dict__ екземпляра).

Наслідування — ієрархія та повторне використання

Базове наслідування + super()
super() повертає проксі-об'єкт, що делегує виклики методів до наступного класу в MRO (Method Resolution Order).
class Animal: def __init__(self, name: str): self.name = name print(f"Animal.__init__: {name}") def speak(self) -> str: raise NotImplementedError class Dog(Animal): def __init__(self, name: str, breed: str): super().__init__(name) # викликаємо батьківський __init__ self.breed = breed def speak(self) -> str: return f"{self.name}: Гав!" class GuideDog(Dog): def speak(self) -> str: base = super().speak() # Dog.speak() return f"{base} (тихіше)" d = GuideDog("Рекс", "Лабрадор") print(d.speak()) # Рекс: Гав! (тихіше) print(isinstance(d, Animal)) # True — ланцюжок isinstance print(issubclass(GuideDog, Dog)) # True
Множинне наслідування + MRO (C3 лінеаризація)
Python визначає порядок пошуку методів за алгоритмом C3. Результат доступний через .__mro__ або mro().
class A: def hello(self): return "A" class B(A): def hello(self): return "B → " + super().hello() class C(A): def hello(self): return "C → " + super().hello() class D(B, C): # Diamond inheritance! pass d = D() print(d.hello()) # B → C → A (не B → A, C → A!) print(D.__mro__) # (D, B, C, A, object) — MRO гарантує кожен клас 1 раз # Правило: super() завжди дивиться на MRO, не на пряму батьківщину!
Множинне наслідування ускладнює код. Надавай перевагу Mixins або Composition. Якщо є сумніви — використовуй issubclass(C, B) для перевірки ієрархії.
Mixin — горизонтальне наслідування
Mixin — клас, що додає поведінку без власного стану. Не використовується самостійно. Ідеальний спосіб уникнути глибокої ієрархії.
class JsonMixin: """Додає JSON серіалізацію будь-якому класу""" def to_json(self) -> str: import json return json.dumps(self.__dict__) @classmethod def from_json(cls, data: str): import json obj = cls.__new__(cls) obj.__dict__.update(json.loads(data)) return obj class LogMixin: """Додає логування методів""" def log(self, msg: str) -> None: print(f"[{self.__class__.__name__}] {msg}") class User(JsonMixin, LogMixin): def __init__(self, name: str, email: str): self.name = name self.email = email u = User("Alice", "a@b.com") print(u.to_json()) # {"name": "Alice", "email": "a@b.com"} u.log("Увійшов у систему") # [User] Увійшов у систему

Поліморфізм — один інтерфейс, різна поведінка

Duck Typing — "якщо крякає як качка..."
Python не перевіряє тип — він перевіряє поведінку. Якщо об'єкт має потрібний метод — він підходить.
class Duck: def quack(self): return "Кря!" def walk(self): return "Ходжу перевалюючись" class Person: def quack(self): return "Я можу крякати!" def walk(self): return "Ходжу нормально" def make_it_quack(duck): # немає type hint на клас! # Python не перевіряє — головне щоб метод існував print(duck.quack()) make_it_quack(Duck()) # Кря! make_it_quack(Person()) # Я можу крякати! make_it_quack("я теж") # AttributeError — рядок не крякає
Перевантаження операторів (operator overloading)
Python дозволяє визначити поведінку стандартних операторів (+, -, *, ==, <, []) для власних класів через magic methods.
from __future__ import annotations from math import sqrt class Vector: def __init__(self, x: float, y: float): self.x, self.y = x, y def __add__(self, other: Vector) -> Vector: return Vector(self.x + other.x, self.y + other.y) def __mul__(self, scalar: float) -> Vector: return Vector(self.x * scalar, self.y * scalar) def __rmul__(self, scalar: float) -> Vector: return self.__mul__(scalar) # 3 * v = v * 3 def __abs__(self) -> float: return sqrt(self.x**2 + self.y**2) def __eq__(self, other: object) -> bool: if not isinstance(other, Vector): return NotImplemented return self.x == other.x and self.y == other.y def __repr__(self) -> str: return f"Vector({self.x}, {self.y})" v1 = Vector(1, 2) v2 = Vector(3, 4) print(v1 + v2) # Vector(4, 6) print(2 * v1) # Vector(2, 4) print(abs(v2)) # 5.0
Завжди повертай NotImplemented (не raise!), якщо тип other не підтримується. Python тоді спробує викликати зворотній метод other.__radd__(self).

Абстракція — інтерфейси та ABC

ABC — Abstract Base Classes
ABC визначає контракт: підклас ЗОБОВ'ЯЗАНИЙ реалізувати всі абстрактні методи. Неможливо створити екземпляр ABC з нереалізованими методами.
from abc import ABC, abstractmethod from typing import List class DataStore(ABC): """Абстрактний інтерфейс для сховища даних""" @abstractmethod def save(self, key: str, value: str) -> None: ... @abstractmethod def load(self, key: str) -> str: ... @abstractmethod def delete(self, key: str) -> None: ... def save_many(self, items: dict) -> None: # Конкретний метод у ABC — може використовувати abstract методи! for k, v in items.items(): self.save(k, v) # поліморфний виклик class RedisStore(DataStore): def save(self, key, value): print(f"Redis: SET {key} {value}") def load(self, key): return f"redis_value_{key}" def delete(self, key): print(f"Redis: DEL {key}") # DataStore() → TypeError: Can't instantiate abstract class store = RedisStore() store.save_many({"a": "1", "b": "2"}) # Redis: SET a 1 / SET b 2
Protocol — структурна типізація (Python 3.8+)
Protocol дозволяє duck typing з підтримкою статичного аналізу (mypy). Клас відповідає Protocol, якщо має потрібні методи — без явного наслідування!
from typing import Protocol, runtime_checkable @runtime_checkable # дозволяє isinstance() перевірку class Drawable(Protocol): def draw(self) -> None: ... def resize(self, factor: float) -> None: ... class Circle: # НЕ наслідує Drawable! def draw(self): print("O") def resize(self, f): self.r *= f class Square: # теж не наслідує def draw(self): print("□") def resize(self, f): self.side *= f def render(shape: Drawable) -> None: # mypy перевіряє Protocol! shape.draw() render(Circle()) # OK — Circle структурно відповідає Drawable print(isinstance(Circle(), Drawable)) # True (runtime_checkable)
Різниця: ABC потребує явного наслідування (is-a). Protocol використовує структурну типізацію (has-a). Protocol ближчий до справжнього duck typing, але з підтримкою mypy.

Магічні методи (Dunder methods) — Python data model

Повний довідник dunder методів
# ── Представлення ───────────────────────────────── __repr__(self) # repr(obj) — для розробника, eval()-able __str__(self) # str(obj) — для користувача, print() __format__(self, spec)# format(obj, spec), f"{obj:spec}" __bytes__(self) # bytes(obj) # ── Порівняння ──────────────────────────────────── __eq__(self, other) # == (не забудь __hash__!) __ne__(self, other) # != (auto якщо визначений __eq__) __lt__(self, other) # < (+ functools.total_ordering) __le__(self, other) # <= __gt__(self, other) # > __ge__(self, other) # >= __hash__(self) # hash(obj), потрібен для set/dict # ── Арифметика ──────────────────────────────────── __add__(self, other) # + __sub__(self, other) # - __mul__(self, other) # * __truediv__ # / __floordiv__ # // __mod__ # % __pow__ # ** __radd__, __rsub__... # зворотні (якщо лівий не знає як) __iadd__, __isub__... # += -= (in-place) __neg__(self) # унарний - __pos__(self) # унарний + __abs__(self) # abs() # ── Контейнери ──────────────────────────────────── __len__(self) # len(obj) __getitem__(self, key)# obj[key] __setitem__(self, k,v)# obj[key] = value __delitem__(self, key)# del obj[key] __contains__(self, x) # x in obj __iter__(self) # iter(obj), for loops __next__(self) # next(obj) __reversed__(self) # reversed(obj) # ── Контекстний менеджер ────────────────────────── __enter__(self) # with obj as x: __exit__(self,tp,v,tb)# після блоку with (навіть при exception) # ── Виклик ──────────────────────────────────────── __call__(self, ...) # obj() — об'єкт поводиться як функція # ── Атрибути ────────────────────────────────────── __getattr__(self, n) # якщо атрибут НЕ знайдений звичайно __getattribute__ # кожен доступ до атрибута (обережно!) __setattr__(self,n,v) # obj.attr = value __delattr__(self, n) # del obj.attr # ── Клас/метаклас ───────────────────────────────── __init_subclass__ # викликається при наслідуванні __class_getitem__ # підтримка Generic[T]
__call__ — callable об'єкти
class Counter: def __init__(self): self.count = 0 def __call__(self, step=1): self.count += step return self.count c = Counter() c() # 1 c() # 2 c(5) # 7 callable(c) # True
__enter__/__exit__ — context manager
class Timer: import time def __enter__(self): self.start = self.time.time() return self # as timer def __exit__(self, *args): elapsed = self.time.time()-self.start print(f"{elapsed:.3f}s") return False # не глушити exc with Timer() as t: x = sum(range(10_000_000))
total_ordering + __hash__ — правильне порівняння
from functools import total_ordering @total_ordering # генерує __le__, __gt__, __ge__ автоматично class Version: def __init__(self, major, minor, patch): self.v = (major, minor, patch) def __eq__(self, other): if not isinstance(other, Version): return NotImplemented return self.v == other.v def __lt__(self, other): if not isinstance(other, Version): return NotImplemented return self.v < other.v def __hash__(self): # ВАЖЛИВО: якщо __eq__ — потрібен __hash__! return hash(self.v) v1 = Version(1, 2, 0) v2 = Version(2, 0, 0) print(v1 < v2) # True print(v1 >= v2) # False (total_ordering!) {v1, v2} # можна в set (є __hash__)
Якщо визначаєш __eq__ без __hash__ — клас стає unhashable (Python встановлює __hash__ = None). Не можна використовувати в set і dict!

Метакласи — клас класів

Як Python створює клас
Метаклас — це "фабрика класів". Так само як клас — фабрика об'єктів, метаклас — фабрика класів. type — дефолтний метаклас.
# Ці два записи ЕКВІВАЛЕНТНІ: class Dog: sound = "Гав" # ↓ Python робить саме це всередині: Dog = type("Dog", (), {"sound": "Гав"}) # type(name, bases, namespace) # Порядок створення класу: # 1. metaclass.__prepare__() → namespace dict # 2. Виконується тіло класу → заповнює namespace # 3. metaclass.__new__() → створює клас-об'єкт # 4. metaclass.__init__() → ініціалізує клас class SingletonMeta(type): """Метаклас, що робить будь-який клас Singleton""" _instances = {} def __call__(cls, *args, **kwargs): if cls not in cls._instances: # super().__call__() = __new__ + __init__ cls._instances[cls] = super().__call__(*args, **kwargs) return cls._instances[cls] class Database(metaclass=SingletonMeta): def __init__(self): self.conn = "active" db1 = Database() db2 = Database() print(db1 is db2) # True — один і той самий об'єкт
__init_subclass__ — легша альтернатива метакласам
Python 3.6+. Викликається автоматично при наслідуванні — ідеально для реєстрації плагінів та валідації підкласів.
class Plugin: _registry: dict = {} def __init_subclass__(cls, name: str = "", **kwargs): super().__init_subclass__(**kwargs) if not name: raise TypeError(f"{cls} потребує name= аргумент") if name in cls._registry: raise ValueError(f"Плагін '{name}' вже існує") cls._registry[name] = cls print(f"Зареєстровано плагін: {name}") class CSVPlugin(Plugin, name="csv"): def process(self, data): ... class JSONPlugin(Plugin, name="json"): def process(self, data): ... print(Plugin._registry) # {'csv': CSVPlugin, 'json': JSONPlugin} plugin = Plugin._registry["csv"]() plugin.process("data")
__class_getitem__ — Generic-підтримка
class TypedList: def __class_getitem__(cls, item): # Повертає спеціалізовану версію класу class SpecializedList(cls): element_type = item def append(self, value): if not isinstance(value, item): raise TypeError(f"Очікується {item}") super().append(value) return SpecializedList IntList = TypedList[int] lst = IntList() lst.append(42) # OK lst.append("str") # TypeError!

Патерни проектування в Python

Singleton
# Через __new__ class Config: _instance = None def __new__(cls): if cls._instance is None: cls._instance = super().__new__(cls) return cls._instance a = Config() b = Config() print(a is b) # True
Factory Method
class Animal: @classmethod def create(cls, kind: str): animals = { "dog": Dog, "cat": Cat, } if kind not in animals: raise ValueError(kind) return animals[kind]() a = Animal.create("dog")
Observer (Publisher/Subscriber)
class EventEmitter: def __init__(self): self._handlers = {} def on(self, event, fn): self._handlers.setdefault(event, []).append(fn) def emit(self, event, *args): for fn in self._handlers.get(event, []): fn(*args) bus = EventEmitter() bus.on("login", lambda u: print(f"Hi {u}")) bus.emit("login", "Alice")
Decorator Pattern
from functools import wraps def retry(times=3): def decorator(fn): @wraps(fn) def wrapper(*a, **kw): for i in range(times): try: return fn(*a, **kw) except Exception: if i == times-1: raise return wrapper return decorator @retry(times=3) def fetch(url): ...
Composition over Inheritance — правильна архітектура
Наслідування — це "є" (Dog is Animal). Композиція — "має" (Car has Engine). Для гнучкої архітектури надавай перевагу композиції.
# ❌ Проблема з наслідуванням: # class FlyingFish(Fish, Bird)? → diamond problem, крихкий код # ✅ Рішення через композицію: class FlyBehavior: def fly(self): print("Лечу!") class SwimBehavior: def swim(self): print("Пливу!") class Duck: def __init__(self): self.fly_behavior = FlyBehavior() self.swim_behavior = SwimBehavior() def fly(self): self.fly_behavior.fly() def swim(self): self.swim_behavior.swim() # Легко змінити поведінку в runtime! duck = Duck() duck.fly_behavior = NoFlyBehavior() # качка з обрізаними крилами

Просунуті концепції

dataclass — автогенерація boilerplate (Python 3.7+)
@dataclass автоматично генерує __init__, __repr__, __eq__ і більше. field() дає точний контроль над кожним полем.
from dataclasses import dataclass, field, KW_ONLY from typing import ClassVar, List @dataclass(frozen=True, order=True) # immutable + сортування class Point: x: float y: float z: float = 0.0 @dataclass class Inventory: name: str count: ClassVar[int] = 0 # не в __init__! items: List[str] = field(default_factory=list) _id: int = field(init=False, repr=False) _: KW_ONLY # наступні — keyword-only max_size: int = 100 def __post_init__(self): Inventory.count += 1 self._id = Inventory.count # обчислення після init inv = Inventory("Склад", max_size=200) p1, p2 = Point(0,0), Point(1,1) print(p1 < p2) # True (order=True) # p1.x = 5 → FrozenInstanceError (frozen=True)
__new__ vs __init__
__new__ створює об'єкт, __init__ ініціалізує. __new__ потрібен для іммутабельних типів та метакласів.
class ImmutablePoint(tuple): def __new__(cls, x, y): # tuple іммутабельний — лише __new__ return super().__new__(cls, (x, y)) def __init__(self, x, y): # tuple вже створено в __new__! self.x = x self.y = y p = ImmutablePoint(3, 4) print(p[0], p.x) # 3 3
__getattr__ vs __getattribute__
class Proxy: def __init__(self, target): # НЕ self.target = target! object.__setattr__(self, '_t', target) def __getattr__(self, name): # Лише якщо attr не знайдений return getattr(self._t, name) def __getattribute__(self, name): # КОЖЕН доступ — небезпечно! # Легко зациклитись тут return object.__getattribute__(self, name)
Weak References та __del__
weakref дозволяє посилатись на об'єкт без збільшення лічильника посилань. Об'єкт буде знищений GC, коли більше немає strong references.
import weakref class Cache: def __init__(self): # Значення зберігаються як weak refs self._data = weakref.WeakValueDictionary() def set(self, key, value): self._data[key] = value def get(self, key): return self._data.get(key) class Resource: def __del__(self): # Фіналайзер — викликається при знищенні # Обережно: не гарантований порядок виклику! print("Ресурс звільнено") cache = Cache() r = Resource() cache.set("key", r) print(cache.get("key")) # <Resource object> del r # "Ресурс звільнено" + cache["key"] тепер None
Уникай __del__ для звільнення ресурсів — використовуй контекстні менеджери (with). __del__ не гарантується при завершенні інтерпретатора та може спричинити циклічні посилання.
MRO та super() у множинному наслідуванні — детально
# Cooperative multiple inheritance — правильний патерн class Base: def __init__(self, **kwargs): # Base повинен поглинати залишкові kwargs super().__init__(**kwargs) # передаємо в object class Timestamped(Base): def __init__(self, created_at=None, **kwargs): self.created_at = created_at super().__init__(**kwargs) # передаємо решту далі по MRO class Tagged(Base): def __init__(self, tags=None, **kwargs): self.tags = tags or [] super().__init__(**kwargs) class Article(Timestamped, Tagged): def __init__(self, title, **kwargs): self.title = title super().__init__(**kwargs) # MRO: Article → Timestamped → Tagged → Base → object a = Article( title="Hello", created_at="2024-01-01", tags=["python", "oop"] ) print(a.title, a.created_at, a.tags) # всі атрибути є!
Правило кооперативного наслідування: кожен __init__ ЗАВЖДИ викликає super().__init__(**kwargs) і ЗАВЖДИ приймає **kwargs. Так super() може коректно пройти весь ланцюжок MRO.
